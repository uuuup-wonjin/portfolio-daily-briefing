## gstack 스킬 설명

### 📋 기획 & 설계
- **/office-hours** — 포트폴리오 아이디어 브레인스토밍 및 전략 수립
- **/plan-ceo-review** — 전체 전략과 범위 검토
- **/design-consultation** — 디자인 시스템 및 UI/UX 계획
- **/design-shotgun** — 비주얼 디자인 탐색

### 🛠️ 개발
- **/plan-eng-review** — 기술 아키텍처 및 구현 계획 검토
- **/autoplan** — 아이디어부터 배포까지 자동 계획 생성
- **/review** — 코드 리뷰 및 품질 검증
- **/qa** — QA 테스트 및 브라우저 검증
- **/ship** — 배포 및 PR 생성

### 🔧 기타
- **/investigate** — 버그 원인 분석 및 디버깅
- **/retro** — 주간 회고 및 학습 정리

## 포트폴리오 개발 워크플로우

```
1️⃣  /office-hours     → 콘셉 정의
2️⃣  /plan-eng-review  → 기술 방향
3️⃣  [코드 작성]        → 개발
4️⃣  /qa               → 검증
5️⃣  /ship             → 배포
```
